## 🎯 テスト実施手順

### Phase 1: 基本アクセス・表示テスト

#### 1.1 サイトアクセステスト
- [ ] トップページ（https://kyoshitsunavi.netlify.app/）にアクセス
- [ ] ページが正常に表示されることを確認
- [ ] ローディング時間を体感的に評価（3秒以内が理想）
- [ ] **開発者コンソール確認**: F12 → Console タブでエラーメッセージを記録

#### 1.2 レスポンシブデザインテスト
**デスクトップ（1920x1080）**
- [ ] 全体のレイアウトが適切に表示
- [ ] ナビゲーションメニューが正常に動作

**タブレット（768x1024）**
- [ ] ブラウザを縮小してタブレットサイズで確認
- [ ] レイアウトが適切に調整される

**スマートフォン（375x667）**
- [ ] 実際のスマートフォンまたはブラウザの開発者ツールで確認
- [ ] ハンバーガーメニューが正常に動作
- [ ] タッチ操作が適切に反応

#### 1.3 基本ページ表示テスト
- [ ] 会社概要ページ（/about）
- [ ] ヘルプページ（/help）
- [ ] プライバシーポリシー（/privacy）
- [ ] 利用規約（/terms）
- [ ] **各ページで開発者コンソール確認**

### Phase 2: 一般ユーザー（生徒・保護者）視点テスト

#### 2.1 教室検索機能テスト
**基本検索**
- [ ] 検索フィールドに何も入力せずに検索ボタンをクリック
- [ ] 全ダミー教室（4件）が表示されることを確認
- [ ] **開発者コンソール確認**: 検索実行時のエラーをチェック

**地域フィルタ検索**
- [ ] 都道府県フィルタを選択して検索
- [ ] 市区町村フィルタを選択して検索
- [ ] 複数の条件を組み合わせて検索

**キーワード検索**
- [ ] 教室名での検索（例: "さくら"）
- [ ] 地域名での検索
- [ ] 存在しないキーワードでの検索

#### 2.2 教室詳細ページテスト
- [ ] 検索結果から教室をクリックして詳細ページにアクセス
- [ ] 教室情報が適切に表示される
- [ ] 料金情報が正しく表示される
- [ ] お問い合わせフォームが表示される
- [ ] **開発者コンソール確認**: 教室詳細ページのエラーをチェック

### Phase 3: 教室運営者視点テスト（本番同様認証フロー）

#### 3.1 新規アカウント登録テスト（推奨）
**注意**: より本番らしいテストのため、新しいメールアドレスでアカウント登録を試すことを推奨します。

- [ ] 新規登録ページにアクセス
- [ ] 有効なメールアドレスを入力
- [ ] パスワードを設定
- [ ] 「教室運営者」を選択
- [ ] 登録ボタンをクリック
- [ ] メール確認リンクが送信されることを確認
- [ ] メールの確認リンクをクリック
- [ ] ログイン成功後、「教室未登録状態」のダッシュボードが表示される
- [ ] **開発者コンソール確認**: 登録プロセス全体でのエラーをチェック

#### 3.2 既存テストアカウントログインテスト
- [ ] ログインページで `test@piano-navi.com` / `test123456` を入力
- [ ] ログイン成功
- [ ] ダッシュボードで「教室未登録」状態が表示される
- [ ] 「教室を登録」ボタンが表示される
- [ ] **開発者コンソール確認**: ログイン時のエラーをチェック

#### 3.3 教室情報登録フロー（本番同様）
- [ ] 「教室を登録」ボタンをクリック
- [ ] 教室情報入力フォームが表示される
- [ ] 必須項目を入力（教室名、住所、電話番号等）
- [ ] 一時保存が可能か確認
- [ ] 登録完了後、ダッシュボードに戻る
- [ ] 教室ステータスが「下書き」または「決済待ち」状態になる
- [ ] **開発者コンソール確認**: 教室登録時のエラーをチェック

#### 3.4 パスワードリセット機能テスト
- [ ] ログインページで「パスワードを忘れた方」リンクをクリック
- [ ] パスワードリセットページが正常に表示される
- [ ] 存在するメールアドレスでリセット要求
- [ ] 存在しないメールアドレスでリセット要求
- [ ] メール送信完了画面が表示される
- [ ] **開発者コンソール確認**: パスワードリセット時のエラーをチェック

#### 3.5 教室情報編集テスト
- [ ] 教室情報編集ページにアクセス
- [ ] 各項目（教室名、住所、電話番号等）を編集
- [ ] 変更を保存
- [ ] 変更が正しく反映されているか確認
- [ ] **開発者コンソール確認**: 保存時のエラーをチェック

#### 3.6 決済機能テスト
- [ ] 掲載料金管理ページにアクセス
- [ ] 月額料金の説明が適切に表示される
- [ ] Stripe決済フォームが正常に表示される
- [ ] **注意**: 実際の決済は行わず、フォーム表示のみ確認
- [ ] **開発者コンソール確認**: 決済ページのエラーをチェック

#### 3.7 ログアウト機能テスト
- [ ] ダッシュボードからログアウト
- [ ] ログイン画面にリダイレクトされる
- [ ] 再度ダッシュボードにアクセスを試行
- [ ] ログイン画面にリダイレクトされることを確認

### Phase 4: 脆弱性・セキュリティテスト

#### 4.1 認証セキュリティテスト
- [ ] ログインせずにダッシュボードURL直接アクセスを試行

#### 5.2 同時操作テスト
